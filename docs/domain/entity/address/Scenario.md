## 📍 Address Concept (Shared Location Model)

### 🧭 Overview

An address represents a real-world physical location used to identify where something exists.

It is not limited to a single type of entity and can be reused across different parts of the system.

---

### 🏷️ Where Address Can Be Used

An address can belong to multiple different business objects such as:

- Seller (business owner or store owner)
- Warehouse (storage location)
- Store / Shop (physical selling point)
- Any other physical location-based entity

---

### 📌 Purpose of Address

The address is used to:

- Identify the exact physical location of an entity
- Support delivery and logistics operations
- Enable users or systems to locate a warehouse or store
- Differentiate between multiple physical locations of the same owner

---

### 🧱 Address Characteristics

An address typically includes:

- Country / City / Region
- Street and building details
- Postal code (if available)
- Optional location coordinates (for mapping systems)

---

### 🔗 Relationship Principle

- Address is a shared concept, not tied to a single entity type
- Multiple entities can reference the same address structure
- Each entity may have its own address or share one depending on the business need

---

### 🚫 Important Notes

- Address must represent a real-world location
- It should be clear and unambiguous enough for physical navigation
- It should not be used for abstract or non-physical entities
